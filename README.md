# GetPrice — منصة مراقبة أسعار الهواتف واكتشاف العروض الحقيقية 🇸🇦

منصة ويب حقيقية قابلة للتشغيل تراقب أسعار الهواتف في المتاجر السعودية
(**Amazon.sa · Jarir · Noon · Extra**)، تحفظ تاريخ السعر، تحلّل الخصومات مقارنةً
بالسعر التاريخي، **تكشف الخصومات الوهمية**، تكتشف العروض القوية تلقائيًا حتى على
هواتف لا يتابعها المستخدم، ثم ترسل **تنبيهًا فوريًا** باسم الهاتف والمتجر والسعر
السابق والحالي ونسبة الخصم الحقيقية ورابط الشراء المباشر.

النظام مبني من ثلاثة أجزاء كما هو مطلوب:

1. **عين تراقب السوق** — `workers/poller.js` يجمع الأسعار دوريًا عبر Adapters مستقلة لكل متجر.
2. **عقل يحلّل** — `services/priceStats.js` + `services/dealAnalysis.js` يحسبان المتوسطات وأدنى/أعلى سعر، الخصم الحقيقي، كشف الخصم الوهمي، و **Deal Score (0–100)**.
3. **صوت ينبّه** — `services/alertEngine.js` + `services/notifier/*` يطلقان التنبيهات عبر Email / Telegram / Web Push / Console مع منع التكرار خلال 24 ساعة.

---

## المتطلبات

- **Node.js 22.5+** (يُفضّل 24). لا حاجة لأي قاعدة بيانات خارجية — نستخدم SQLite المدمجة في Node (`node:sqlite`).
- لا حاجة لأدوات بناء native (لا `better-sqlite3`، لا مترجم C++).

## التشغيل خطوة بخطوة

```bash
# 1) تثبيت الحزم
npm install

# 2) إعداد المتغيرات (انسخ المثال وعدّله حسب الحاجة)
cp .env.example .env

# 3) توليد بيانات تجريبية (كتالوج هواتف + 4 متاجر + تاريخ 90 يومًا + مستخدم تجريبي)
npm run seed

# 4) تشغيل الخادم
npm start
# افتح: http://localhost:3000
```

**مستخدم تجريبي جاهز:** `demo@getprice.local` / كلمة المرور `demo1234`

### أوامر مساعدة

```bash
npm run poll   # تشغيل دورة جمع أسعار واحدة يدويًا + تقييم التنبيهات
npm run scan   # فحص العروض القوية يدويًا + إطلاق التنبيهات التلقائية
npm run dev    # تشغيل مع إعادة التحميل التلقائي (node --watch)
```

الجدولة تعمل تلقائيًا داخل الخادم عبر `node-cron` (انظر `POLL_CRON` و `SCAN_CRON`).

## مصدر الأسعار: Seed مقابل Scraper

يتحكم `PRICE_SOURCE` في `.env`:

- `seed` (افتراضي) — يولّد سعرًا حاليًا واقعيًا محليًا. **آمن للتجربة بدون إنترنت** ويوفّر مسارًا حقيقيًا لجلب السعر وتخزينه.
- `scraper` — يفعّل الزحف الحقيقي: كل adapter يجلب صفحة المنتج ويستخرج السعر عبر `cheerio`، مع:
  - احترام **Rate Limit** (`SCRAPE_MIN_DELAY_MS` بين طلبات نفس المتجر).
  - `User-Agent` معرّف (`SCRAPE_USER_AGENT`).
  - مراقبة فشل لكل adapter + تخزين آخر سعر ناجح + تنبيه إداري بعد فشل متكرر.

> ⚠️ **ملاحظة تشغيلية:** لا يمكن ضمان استمرار عمل الـ Scraping للأبد (قد تُغيّر المتاجر تصميمها أو تمنع الطلبات الآلية). لذلك النظام مصمّم بـ **Adapter Pattern**: منطق كل متجر معزول في ملف مستقل (`src/adapters/*.js`)، وإضافة **API رسمي أو Affiliate API** لاحقًا = ملف adapter جديد يُسجَّل في `src/adapters/index.js` دون تغيير باقي النظام. المحدّدات (selectors) في ملفات الـ adapters تقريبية وقد تحتاج تحديثًا للزحف الفعلي.

**احترم دائمًا شروط استخدام المتاجر و `robots.txt`.** استخدم APIs الرسمية عند توفّرها.

## كيف يُكشف الخصم الوهمي؟ (المنطق الأساسي)

لا نعتمد على السعر المشطوب المعلن. في `services/dealAnalysis.js`:

- **الخصم الحقيقي** = أكبر انخفاض مقابل *متوسط آخر 30 يومًا* و *متوسط آخر 90 يومًا* (وليس مقابل السعر المشطوب).
- يُصنَّف العرض **مشكوكًا فيه** إذا كان الخصم المعلن كبيرًا بينما الخصم الحقيقي مقابل المتوسط التاريخي ضعيف، أو عند وجود تذبذب يوحي برفع السعر ثم خفضه.
- **مثال حقيقي من البيانات التجريبية:** جهاز عليه خصم معلن 39.8% لكن الخصم الحقيقي مقابل متوسط 30 يومًا 5.7% فقط → يُصنَّف «مشكوك فيه».

### نموذج Deal Score (0–100)

| الوزن | المعيار |
|------|---------|
| 35 | نسبة الانخفاض مقابل متوسط 30 يومًا (40%+ = كامل النقاط) |
| 25 | هل السعر الحالي هو أقل سعر تاريخي؟ |
| 15 | موثوقية المتجر وتوفّر المنتج |
| 15 | استقرار الانخفاض (وليس تذبذبًا يوميًا) |
| 10 | توافق الهاتف مع تفضيلات المستخدم |

التصنيف: **85–100 ممتاز · 70–84 قوي · 50–69 جيد · <50 عادي**.

### قواعد إطلاق التنبيه

يُرسل تنبيه إذا تحقق شرط واحد على الأقل:

- السعر الحالي ≤ السعر المستهدف للمستخدم.
- السعر الحالي = أقل سعر تاريخي.
- الخصم الحقيقي ≥ `ALERT_REAL_DISCOUNT_PCT` (افتراضي 40%) مقابل متوسط 30 يومًا.
- Deal Score ≥ `ALERT_MIN_DEAL_SCORE` (افتراضي 70).
- المنتج يطابق تفضيلات المستخدم (اكتشاف تلقائي) حتى لو لم يكن في قائمة المتابعة.

**منع التكرار:** لا يُرسَل نفس التنبيه لنفس المستخدم/المنتج/شريحة السعر أكثر من مرة خلال 24 ساعة.

## قنوات التنبيه

تُضبط لكل مستخدم من صفحة الإعدادات (CSV):

| القناة | الإعداد المطلوب في `.env` |
|-------|--------------------------|
| **Email** | `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `MAIL_FROM` |
| **Telegram** | `TELEGRAM_BOT_TOKEN` + `telegram_chat_id` للمستخدم |
| **Web Push** | `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY` (ولّدها: `npx web-push generate-vapid-keys`) |
| **Console** | يعمل دائمًا — يطبع التنبيه في سجل الخادم (مفيد للتجربة) |

أي قناة غير مُعدّة تسقط تلقائيًا إلى **Console** حتى لا يُفقد التنبيه.
تنبيهات فشل الـ adapters تُرسل إلى `ADMIN_EMAIL` / `TELEGRAM_ADMIN_CHAT_ID`.

## الواجهات

- **الرئيسية** `/` — شرح الفكرة + بحث + أحدث العروض القوية.
- **العروض** `/deals` — كل العروض المكتشفة تلقائيًا مع فلترة (متجر/شركة/خصم/سعر/ذاكرة) وترتيب حسب قوة العرض.
- **صفحة الهاتف** `/phone/:slug` — الأسعار في كل متجر، أقل/أعلى سعر، متوسط 7/30/90، **رسم بياني لتاريخ السعر**، وتحليل «هل يستحق الشراء؟».
- **لوحتي** `/dashboard` — الهواتف المتابَعة + حالة كل عرض + سجل التنبيهات.
- **إضافة** `/add` — إضافة هاتف بالاسم أو **رابط منتج مباشر**.
- **الإعدادات** `/settings` — القنوات، الحد الأدنى للخصم، الحد الأعلى للسعر، الشركات المفضّلة.

## بنية المشروع

```
src/
  server.js               # نقطة الدخول (Express) + جدولة الـ workers
  config.js               # قراءة .env
  db/
    index.js              # اتصال node:sqlite + المخطط (هجرات idempotent)
    models.js             # طبقة الوصول للبيانات
    seed.js               # بيانات تجريبية
  adapters/               # Adapter Pattern — متجر واحد لكل ملف
    base.js  amazonSa.js  jarir.js  noon.js  extra.js  index.js
  services/
    priceStats.js         # متوسطات 7/30/90 + أدنى/أعلى + استقرار
    dealAnalysis.js       # الخصم الحقيقي + كشف الوهمي + Deal Score
    alertEngine.js        # قواعد التنبيه + منع التكرار
    notifier/             # email · telegram · webpush · console
    phoneView.js  catalog.js
  workers/
    poller.js             # جمع الأسعار الدوري
    dealScanner.js        # اكتشاف العروض التلقائي
  routes/  auth.js  main.js  api.js
  auth/    middleware.js  sqliteSessionStore.js
  views/   *.ejs          # الواجهات
  public/  css/ js/        # الأنماط + الرسم البياني + Service Worker
```

## الجودة

- معالجة أخطاء واضحة + صفحات 404/خطأ.
- **Logging** مع طوابع زمنية لكل worker وخدمة.
- كلمات المرور مُخزّنة كـ **bcrypt hash** (لا نص صريح).
- احترام **Rate Limits** في الزحف.
- `.env.example` + بيانات **Seed** جاهزة.
- مراقبة صحة الـ adapters عبر `/api/health/adapters` وجدول `poll_runs`.

## معيار النجاح ✅

المستخدم يستطيع: إنشاء حساب → إضافة هاتف/رابط → رؤية السعر وتاريخه → تحديد سعر
مستهدف → استقبال تنبيه عند الانخفاض → رؤية عروض قوية مكتشفة تلقائيًا → معرفة إن كان
الخصم حقيقيًا أم وهميًا → الضغط على رابط الشراء المباشر.
